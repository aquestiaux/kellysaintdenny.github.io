Setup Instructions

‚ö†Ô∏è Important: These setup steps only need to be completed once. After setup, you'll have instant access to your secure CMS.

Enable GitHub OAuth

First, we need to authorize the CMS to access your GitHub repository securely:

  1. Go to GitHub Developer Settings
  2. Click "New OAuth App"
  3. Fill in the following details:
OAuth App Configuration
Application name: Kelly Saint Denny CMS
Homepage URL: https://kellysaintdenny.com
Authorization callback URL: https://kellysaintdenny.com/admin/cms/index.html

After creating the OAuth app, you'll get a Client ID and Client Secret. Save these securely - you'll need them in the next step.

Configure Netlify Identity

Since you're using GitHub Pages, we'll use Netlify's free identity service for authentication:

  1. Go to Netlify Dashboard
  2. Click "Add new site" ‚Üí "Import an existing project"
  3. Connect your GitHub account and select kellysaintdenny.github.io
  4. In Site settings ‚Üí Identity, click "Enable Identity"
  5. Go to Identity ‚Üí Settings ‚Üí Registration
  6. Set "Registration preferences" to "Invite only"
  7. Go to Identity ‚Üí Settings ‚Üí External providers
  8. Enable GitHub and enter your OAuth credentials from Step 1

Update CMS Configuration

Now we need to update the CMS configuration with your Netlify site details:

Update admin/config.yml
backend:
  name: git-gateway
  branch: main
  base_url: https://YOUR_NETLIFY_SITE.netlify.app

local_backend: false

# ... rest of the configuration stays the same

Replace YOUR_NETLIFY_SITE with your actual Netlify site URL (e.g., amazing-kelly-cms.netlify.app).

Add Netlify Identity Widget

Add the Netlify Identity widget to your admin pages:

Add to admin/cms.html (before closing </head>)
<script src="https://identity.netlify.com/v1/netlify-identity-widget.js"></script>
Add to admin/cms.html (before closing </body>)
<script>
  if (window.netlifyIdentity) {
    window.netlifyIdentity.on("init", user => {
      if (!user) {
        window.netlifyIdentity.on("login", () => {
          document.location.href = "/admin/cms/";
        });
      }
    });
  }
</script>

Test Your Setup

Now it's time to test your secure CMS:

  1. Visit https://kellysaintdenny.com/admin/cms.html
  2. Click "Open CMS"
  3. Sign in with your GitHub account
  4. You should see the professional CMS interface
  5. Try creating a test blog post

Using the CMS

  4. Add tags and categories for organization
  5. Set the language (English or French)

  • Editing: Click any existing post to edit it

  • Version Control: All changes are tracked in Git

Troubleshooting

Common Issues & Solutions:

Authentication Issues

  • Can't sign in: Check that GitHub OAuth is properly configured
  • Permission denied: Ensure your GitHub account has access to the repository
  • OAuth error: Verify callback URLs match exactly

Content Not Publishing

  • Check Git status: Ensure changes are committed to the main branch
  • GitHub Pages: Verify GitHub Pages is enabled and pointing to main branch
  • Build errors: Check GitHub Actions for any build failures
